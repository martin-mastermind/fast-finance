# Plan: AI Money-Saving Assistant Feature

## Overview
Add an AI assistant to Fast Finance that analyzes user spending patterns and provides personalized money-saving tips, insights, and savings goals tracking.

## Current State Analysis
- **Project**: Fast Finance - Telegram Mini App for personal finance
- **Stack**: Bun + ElysiaJS (backend), Next.js + Tailwind (frontend), PostgreSQL + Drizzle ORM
- **Existing Features**: Users, Accounts, Categories, Transactions, Currency conversion, Telegram Bot

## Implementation Steps

### 1. Database Schema Updates
**File**: `packages/db/src/schema.ts`
- Add `savings_goals` table: userId, name, targetAmount, currentAmount, deadline, currency, createdAt
- Add `ai_insights` table: userId, type (tip/insight/alert), title, message, data (JSON), isRead, createdAt
- Add `ai_chat_messages` table: userId, role (user/assistant), content, metadata (JSON), createdAt
- Export new types: SavingsGoal, AiInsight, AiChatMessage

### 2. Backend AI Service
**File**: `apps/backend/src/domain/ai.service.ts`
- `analyzeSpending(userId, period)` - Analyze spending patterns by category, trends
- `generateSavingsTips(userId)` - Generate personalized tips based on spending analysis
- `detectOverspending(userId)` - Alert on unusual spending patterns
- `calculatePotentialSavings(userId)` - Estimate possible savings based on patterns
- `trackSavingsGoalProgress(userId, goalId)` - Monitor goal achievement
- `callLLM(prompt, context)` - Interface to LLM (OpenAI/Groq) for insights

### 3. Backend AI Routes
**File**: `apps/backend/src/routes/ai.ts`
- POST `/ai/chat` - Chat with AI assistant
- GET `/ai/insights` - Get personalized insights
- GET `/ai/tips` - Get money-saving tips
- POST `/ai/goals` - Create savings goal
- GET `/ai/goals` - List user's savings goals
- PATCH `/ai/goals/:id` - Update goal progress
- DELETE `/ai/goals/:id` - Delete savings goal
- POST `/ai/insights/:id/read` - Mark insight as read

### 4. Update Main Backend Index
**File**: `apps/backend/src/index.ts`
- Import and use `aiRouter`
- Add to Elysia app chain

### 5. Frontend API Client Updates
**File**: `apps/frontend/lib/api.ts`
- Add `ai` namespace to ApiClient:
  - `chat(message)` - Send chat message to AI
  - `getInsights()` - Fetch insights
  - `getTips()` - Fetch tips
  - `createGoal(data)` - Create savings goal
  - `getGoals()` - List goals
  - `updateGoal(id, data)` - Update goal
  - `deleteGoal(id)` - Delete goal
  - `markInsightRead(id)` - Mark insight as read

### 6. Frontend AI Store
**File**: `apps/frontend/store/ai.ts`
- State: messages, insights, goals, isLoading, unreadCount
- Actions:
  - `sendMessage(content)` - Send chat message
  - `loadInsights()` - Load insights
  - `loadGoals()` - Load savings goals
  - `createGoal(data)` - Create new goal
  - `updateGoalProgress(id, amount)` - Update goal
  - `markInsightRead(id)` - Mark as read

### 7. AI Assistant Component
**File**: `apps/frontend/components/ai-assistant.tsx`
- Chat interface with AI assistant
- Display spending insights and tips
- Savings goals management (create, view, update)
- Quick actions: "Analyze my spending", "Set a goal", "Get tips"
- Animated UI with Framer Motion
- Responsive design matching app theme

### 8. Update Dashboard
**File**: `apps/frontend/components/dashboard.tsx`
- Add AI assistant tab to BottomNav
- Show AI insights preview card on dashboard
- Display unread insights badge

### 9. Update Finance Store
**File**: `apps/frontend/store/finance.ts`
- Add `'assistant'` to activeTab options
- Add `isAiModalOpen` state (optional)

### 10. Environment Variables
**File**: `.env.example`
- `GROQ_API_KEY` - Groq API key (free, get at console.groq.com)
- `AI_MODEL` - Model name (default: llama-3.1-8b-instant)
- `AI_ENABLED` - Enable/disable AI features (default: true)
- `AI_RATE_LIMIT_PER_USER` - Max requests per user per minute (default: 6)

### 11. Database Migrations

- Run migrations for existing deployments

## Technical Decisions

### LLM Integration (FREE Options)
**Recommended: Groq with Llama 3.1 8B (BEST FREE OPTION)**
- **Free tier**: 14,400 requests/day, 6,000 tokens/minute
- **Sign up**: https://console.groq.com (just email, no credit card)
- **Model**: `llama-3.1-8b-instant` (fast, good for financial advice)
- **Cost**: $0 for this usage level

**Alternative FREE Options:**
1. **Google AI Studio** - 20-500 requests/day depending on model (no credit card)
2. **Cerebras** - 14,400 requests/day (Llama 3.1 8B)
3. **OpenRouter** - 50 requests/day (multiple free models)

**Implementation Strategy:**
- Primary: Groq Llama 3.1 8B
- Fallback: Rule-based analysis if API fails
- Caching: Cache spending analysis for 1 hour
- Rate limiting: Max 1 request per 10 seconds per user (prevents abuse)

**Cost Calculation:**
- Assuming 100 users, each making 10 AI requests/day = 1,000 requests/day
- Groq free tier allows 14,400/day → plenty of headroom
- No cost until you exceed limits

### Privacy
- Only send anonymized spending patterns to LLM (category names + amounts, no personal data)
- Store chat history locally per user
- Allow users to disable AI features

### Performance
- Async insight generation (don't block user)
- Lazy-load AI component
- Debounce chat messages (300ms)
